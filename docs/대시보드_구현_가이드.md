# 대시보드 구현 가이드 (목업)

## 1. 사용 라이브러리 및 UI

### 차트
- **Recharts** (`recharts`)
  - React 기반 차트 라이브러리로, 선 그래프(LineChart), 원형 차트(PieChart) 등 사용.
  - `ResponsiveContainer`로 반응형 대응.
  - 대안: Chart.js, ApexCharts, Highcharts, D3.js 등.

### UI·스타일
- **Tailwind CSS** (v4): 유틸리티 기반 스타일, 반응형(grid, breakpoints) 적용.
- 레퍼런스 이미지와 유사한 카드·사이드바·버튼은 Tailwind만으로 구현.

### 추천 조합 (확장 시)
- **UI 컴포넌트**: Material-UI(MUI), Ant Design, shadcn/ui 등으로 테이블·모달·폼 통일.
- **폼/검증**: React Hook Form + Zod.

## 2. 반응형 및 모바일(플러터) 연동

### 웹(Next.js) 반응형
- **Tailwind**: `sm:`, `md:`, `lg:` breakpoint로 카드/차트 그리드 조정.
- **Recharts**: `ResponsiveContainer`로 부모 너비에 맞춰 차트 크기 자동 변경.
- **사이드바**: 작은 화면에서는 햄버거 메뉴로 접었다 펼치는 방식으로 확장 가능.

### 웹 ↔ 플러터 관계 (동일 구현 가능 여부)

**결론: 화면·기능·데이터는 동일하게 구현 가능하지만, 라이브러리·코드·반응형 구현 방식은 웹과 플러터가 완전히 다릅니다.**

| 구분 | 웹 (현재 구현) | 모바일 (플러터) |
|------|----------------|-----------------|
| 프레임워크 | Next.js (React) | Flutter (Dart) |
| 차트 | Recharts | fl_chart, syncfusion_flutter_charts 등 |
| 스타일/반응형 | Tailwind, ResponsiveContainer | Flutter 위젯, LayoutBuilder, MediaQuery |
| 코드 공유 | ❌ 불가 | ❌ 불가 |

- **동일하게 가능한 것**: 화면 구성(통합 대시보드, 실시간 모니터링 등), 메뉴 구조, 표시 데이터(같은 API 사용), 반응형 개념(화면 크기에 맞춰 레이아웃 조정).
- **동일하지 않은 것**: 웹용 라이브러리(Recharts, Tailwind 등)와 반응형 코드는 플러터에서 그대로 쓸 수 없음. 플러터는 별도 코드베이스에서 Flutter 전용 라이브러리로 같은 UX를 구현.

- **공유하는 것**: 백엔드 API, 데이터 스키마, 화면/메뉴 기획(문서화).
- **플러터 측 구현**:
  - 차트: `fl_chart`, `syncfusion_flutter_charts`, `charts_flutter` 등 플러터 전용 라이브러리 사용.
  - UI: Flutter 위젯으로 동일한 메뉴와 화면 구성을 **새로** 구현.
- **체크 포인트**:
  - API 스펙(엔드포인트, 요청/응답 형식)을 문서화해 웹·플러터가 동일하게 사용.
  - 대시보드 지표·기간·필터 등을 API 파라미터로 통일하면, 웹과 앱에서 같은 데이터를 각 플랫폼에 맞는 UI로 표시할 수 있습니다.

## 3. 폴더 구조

```
frontend/
├── app/
│   ├── layout.tsx              # 루트 레이아웃 (사이드바 + 메인 영역)
│   ├── page.tsx                # / → /dashboard 리다이렉트
│   ├── globals.css
│   ├── dashboard/
│   │   └── page.tsx            # 통합 대시보드 (/dashboard)
│   ├── monitoring/
│   │   └── page.tsx            # 실시간 모니터링 (/monitoring)
│   ├── alerts/
│   │   └── page.tsx            # 알림 (/alerts)
│   ├── qr/
│   │   └── page.tsx            # QR 조회 (/qr)
│   ├── reports/
│   │   └── page.tsx            # 자동 리포트 (/reports)
│   └── login/
│       └── page.tsx            # 로그인 (/login)
├── components/
│   ├── layout/
│   │   ├── Sidebar.tsx         # 좌측 메뉴 (6개 항목)
│   │   └── DashboardLayout.tsx # 사이드바 + children (루트 layout에서 사용)
│   └── dashboard/
│       ├── SummaryCard.tsx           # 요약 카드
│       ├── TrafficChart.tsx          # Traffic 선 그래프
│       ├── UsageChart.tsx            # Usage 선 그래프
│       ├── TrafficTrendChart.tsx     # 상위 5개 앱 트렌드
│       └── ProductiveMonitoringChart.tsx # 원형 차트
├── docs/
│   └── 대시보드_구현_가이드.md
├── public/
├── package.json
├── tsconfig.json
└── next.config.ts
```

- **진입 경로**: `/` 접속 시 `/dashboard`로 리다이렉트되어 **통합 대시보드**가 처음 선택된 화면으로 표시됩니다.
