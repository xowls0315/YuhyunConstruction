## 프론트엔드 인수인계 요약 (유현건설 계측 모니터링)

이 문서는 현재까지 구현된 **웹 프론트엔드(Next.js)** 상태를 정리해, 이후 담당자가 개발을 이어받을 수 있도록 하기 위한 요약본입니다.  
백엔드는 아직 연동하지 않았고, **모든 화면은 목업/퍼블리싱 단계**입니다.

---

### 1. 기술 스택 / 구조

- **Framework**: Next.js 16 (App Router, `app/` 디렉터리)
- **언어**: TypeScript + React 19
- **스타일**: Tailwind CSS v4 (`app/globals.css`에서 import)
- **차트**: Recharts (`recharts`)
- **글꼴**: Pretendard Variable (`public/PretendardVariable.woff2`)

기본 구조는 `create-next-app` 기반이며, 프론트엔드 코드는 `YuhyunConstruction/frontend` 폴더 안에 있습니다.

---

### 2. 라우팅 구조

`app/` 디렉터리 기준 주요 페이지는 다음과 같습니다.

- `/` → `app/page.tsx`  
  - `redirect("/dashboard")` 로 항상 **통합 대시보드**가 첫 화면으로 열리도록 설정.
- `/dashboard` → `app/dashboard/page.tsx`  
  - 통합 대시보드 화면 (그래프/카드 구성).
- `/monitoring` → `app/monitoring/page.tsx`  
  - 실시간 모니터링 목업 화면 (센서 테이블).
- `/alerts` → `app/alerts/page.tsx`  
  - 알림 목록 및 요약 카드.
- `/qr` → `app/qr/page.tsx`  
  - QR 코드/센서 ID 조회 폼 + 결과 영역 목업.
- `/reports` → `app/reports/page.tsx`  
  - 자동 리포트 목록(일보/주보/월보) 목업.
- `/login` → `app/login/page.tsx`  
  - 로그인 폼 목업 (아이디/비밀번호 입력만 존재, 실제 인증 없음).

모든 페이지는 공통 레이아웃을 사용합니다.

- `app/layout.tsx`  
  - 글로벌 CSS 로드 + `<DashboardLayout>{children}</DashboardLayout>` 래핑.
- `components/layout/DashboardLayout.tsx`  
  - 좌측 사이드바 + 우측 콘텐츠 레이아웃.
  - 모바일에서는 상단 햄버거 버튼 + 오버레이로 사이드바를 토글.
- `components/layout/Sidebar.tsx`  
  - 메뉴 6개
    - 통합 대시보드 (`/dashboard`)
    - 실시간 모니터링 (`/monitoring`)
    - 알림 (`/alerts`)
    - QR 조회 (`/qr`)
    - 자동 리포트 (`/reports`)
    - 로그인 (`/login`)
  - 현재 메뉴는 SVG path 기반 아이콘 + Tailwind 스타일로 구성됨.

---

### 3. 페이지별 구현 내용 (목업)

#### 3.1 통합 대시보드 (`/dashboard`)

파일: `app/dashboard/page.tsx`  
사용 컴포넌트: `SummaryCard`, `TrafficChart`, `UsageChart`, `TrafficTrendChart`, `ProductiveMonitoringChart`

- 상단 요약 카드 4개
  - 당일 트래픽 / 당일 사용량 / 최근 30일 구독 / 배포된 애플리케이션
- 상단 그래프 2개
  - 전체 앱 트래픽 발생량 (LineChart)
  - 전체 앱 사용량 (LineChart)
- 하단 그래프 2개
  - 상위 5개 사용 앱 트래픽 추이 비교 (LineChart)
  - 운영 모니터링 (PieChart)
- 하단 각주
  - 모니터링 데이터 반영 지연, 트래픽 대시보드 안내 문구 (한글로 변경 완료).
- 모든 데이터는 **하드코딩된 더미 데이터**이며, 추후 API 연동 필요.

#### 3.2 실시간 모니터링 (`/monitoring`)

파일: `app/monitoring/page.tsx`  
사용 컴포넌트: `SummaryCard`

- 상단 요약 카드 4개
  - 연동 센서 수 / 정상 센서 / 주의 센서 / 경고 센서
- 센서 목록 테이블
  - 컬럼: 센서 ID, 센서 타입, 측정값, 상태(정상/주의), 마지막 업데이트
  - 5개의 가상 행을 반복 렌더링 (실제 데이터 아님).
- 하단 안내 문구
  - `* 실제 API 연동 시 센서 데이터가 표시됩니다`

#### 3.3 알림 (`/alerts`)

파일: `app/alerts/page.tsx`  
사용 컴포넌트: `SummaryCard`

- 상단 요약 카드 3개
  - 미확인 알림 / 오늘 알림 / 이번 주 알림
- 최근 알림 목록
  - 레벨(주의/정보/정상), 메시지, 시간, 확인 버튼으로 구성된 카드 리스트.
- 하단 안내 문구
  - `* 실제 API 연동 시 경보 이벤트가 표시됩니다`

#### 3.4 QR 조회 (`/qr`)

파일: `app/qr/page.tsx`

- 좌측: QR 코드/센서 ID 입력 폼
  - 텍스트 입력 필드 + 조회 버튼.
- 우측: 조회 결과 영역
  - 현재는 안내 문구와 `(목업 화면)` 텍스트만 표시, 실제 데이터 없음.

#### 3.5 자동 리포트 (`/reports`)

파일: `app/reports/page.tsx`

- 상단 리포트 타입 필터 버튼
  - 일보/주보/월보 (실제 필터 동작은 없음, 스타일만 존재).
- 리포트 목록 테이블
  - 생성일 / 리포트 타입 / 기간 / 다운로드(PDF 버튼) 컬럼.
  - 몇 개의 더미 행 하드코딩.
- 하단 안내 문구
  - `* 실제 API 연동 시 생성된 리포트 목록이 표시됩니다`

#### 3.6 로그인 (`/login`)

파일: `app/login/page.tsx`

- 아이디/비밀번호 폼
  - 입력 필드 + 로그인 버튼.
  - `type=\"button\"` 으로 설정되어 있으며, 실제 요청/검증 로직 없음.
- 하단 안내 문구
  - `* 실제 API 연동 시 인증 기능이 적용됩니다`

### 4. 공통 컴포넌트 / 스타일

#### 4.1 레이아웃 & 사이드바

- `components/layout/DashboardLayout.tsx`
  - 모바일:
    - 상단 56px 높이 바 + 햄버거 버튼.
    - 버튼 클릭 시 `sidebarOpen` 상태를 true로 하여 사이드바 슬라이드 인.
    - 검은 오버레이 클릭 시 닫힘.
  - 데스크톱:
    - 좌측 고정 사이드바 + 우측 스크롤 영역.
- `components/layout/Sidebar.tsx`
  - `fixed` (모바일), `md:relative` (데스크톱) 로 동작.
  - `translate-x` 전환으로 슬라이드 인/아웃.
  - 현재 활성 라우트는 `pathname` 기반으로 색상 변경.

#### 4.2 대시보드 컴포넌트

- `SummaryCard.tsx`
  - 라벨 + 값 표시용 카드 (다양한 페이지에서 재사용).
- 차트 컴포넌트 (Recharts)
  - `TrafficChart.tsx` / `UsageChart.tsx` / `TrafficTrendChart.tsx` / `ProductiveMonitoringChart.tsx`
  - 모두 더미 데이터 + 한글 라벨/범례로 세팅됨.
  - 높이 고정: `h-[280px]`.

#### 4.3 글로벌 스타일

- `app/globals.css`
  - Tailwind v4 import.
  - Pretendard Variable `@font-face` 설정 (100~900 가변).
  - `@theme` 로 `--font-sans`를 Pretendard로 지정 (Tailwind 유틸 전역 적용).

자세한 색상/타이포/간격 등은 `docs/DESIGN_SYSTEM_Flutter.md`에 정리되어 있습니다.

### 5. 아직 구현되지 않은 부분 (TODO)

다음 항목은 **향후 담당자가 구현해야 하는 부분**입니다.

1. **백엔드 API 연동**
   - 대시보드 차트/요약 카드 → 실제 계측 데이터 API 연결.
   - 실시간 모니터링 테이블 → 실시간/최근 데이터 조회 API.
   - 알림 목록 → 경보 이벤트 API.
   - 리포트 목록/다운로드 → 리포트 생성·목록·다운로드 API.
2. **인증/권한**
   - 로그인 페이지 → 실제 로그인 API 연동, 토큰/세션 관리, 보호된 페이지 처리.
3. **에러/로딩 상태**
   - 로딩 스피너, 에러 메시지, 빈 상태(데이터 없음) 처리.
4. **필터/검색 기능**
   - 기간 필터, 센서 타입 필터, 검색 입력값을 API 파라미터와 연결.
5. **국제화/다국어**
   - 현재는 전부 한국어 텍스트. 향후 i18n 도입 시 텍스트 추출 필요.

### 6. 실행 / 배포 관련

- **로컬 실행**
  - 루트: `YuhyunConstruction/frontend`
  - 명령어:
    - `npm install`
    - `npm run dev` → `http://localhost:3000`
- **빌드**
  - `npm run build` (현재까지 빌드 성공 확인).
- **Vercel 배포**
  - GitHub 연동 후, Vercel 프로젝트의 **Root Directory** 를 `YuhyunConstruction/frontend` 로 설정해야 함.
  - 자세한 내용은 (필요 시) 별도 배포 가이드 문서로 추가 가능.

---

### 7. 인수인계 시 전달하면 좋은 포인트

- 현재 단계는 **UI 목업**이므로, 백엔드 연동 시 마크업과 구조를 크게 바꾸지 않고 데이터만 바꾸는 방향을 권장.
- 차트/테이블/폼 등 주요 UI 패턴이 이미 잡혀 있으니, **API 응답 스키마에 맞춰 타입 정의 → 더미 데이터 대체** 방식으로 작업하면 됨.
- Flutter 팀과는 `docs/DESIGN_SYSTEM_Flutter.md` 와 `docs/대시보드_구현_가이드.md` 를 공유하면, 웹·앱 간 디자인 일관성을 유지하기 좋습니다.
